    ARIMA and SARIMA Foundations

    1. Overview

    This note summarizes core ideas, modeling assumptions, and practical use cases for the topic.

    2. Core Model

    $$\phi(B)(1-B)^dy_t=\theta(B)\varepsilon_t$$

    Interpretation:

    1. Define the decision variables and constraints clearly.
    2. Separate identification assumptions from estimation choices.
    3. Report sensitivity to key hyperparameters or priors.

    3. Practical Workflow

    1. Start with a baseline model and diagnostics.
    2. Add complexity only when it improves out-of-sample behavior.
    3. Use rolling evaluation and failure-case analysis.

    4. Typical Failure Modes

    1. Data leakage across time or entities.
    2. Mis-specified objective under policy constraints.
    3. Overfitting to benchmark datasets.

    5. Implementation Notes

    1. Keep preprocessing and feature windows reproducible.
    2. Log model version, data snapshot, and seed.
    3. Prefer interpretable checkpoints for production handoff.

    6. Further Reading

    ADF/KPSS, ACF/PACF, Ljung-Box diagnostics.
