    3 五个最值得复用的建模机制

    3.1 双重边际化与协调契约

    经典问题：上游和下游分别加价，导致系统总利润低于一体化最优。常见修复是两部收费、收入分享或 buyback 合同。

    3.2 信息不对称下的筛选博弈

    供应商不知道零售商类型（需求/成本/服务能力）时，需要通过菜单合约实现自选择：

    $$\max_{{w(q),q}} \mathbb{E}[\Pi_M] \quad \text{s.t. IC, IR}$$

    IC（激励相容）和 IR（个体理性）是论文中反复出现的约束框架。

    3.3 网络效应与平台补贴

    平台竞争常见于双边市场，关键在于交叉网络效应：

    $$U_A = v_A - p_A + \alpha N_B, \quad U_B = v_B - p_B + \beta N_A$$

    补贴并非越多越好，最优补贴取决于跨边弹性与进入时序。

    3.4 动态定价与战略等待

    战略消费者会等待降价，形成“当前利润 vs 未来需求”的动态权衡：

    $$V_t(s)=\max_{p_t}{\pi(p_t,s)+\delta \mathbb{E}[V_{t+1}(s’)|p_t,s]}$$

    这类模型对票务、酒店、云资源和广告竞价都可迁移。

    3.5 算法代理与竞争治理

    当企业使用同类外包定价算法时，可能出现“无显式合谋但结果趋同”的机制风险，这为平台监管与反垄断研究提供了新方向。

    4 论文阅读模板（你可以直接复用）

    建议每篇按以下六步做结构化笔记：

    1. 市场结构：参与者、时序、信息结构
    2. 策略空间：价格、产量、服务、开放度、补贴等
    3. 均衡概念：Nash / Stackelberg / PBE / 演化稳定策略
    4. 关键定理：存在性、唯一性、比较静态
    5. 管理含义：哪些参数最可控、哪些结论可执行
    6. 外推边界：哪些现实条件下会失效
